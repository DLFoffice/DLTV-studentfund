## 🔥 การย้ายข้อมูลไป Firebase (Firestore)

### ขั้นที่ 1 — สร้างโปรเจกต์ Firebase
1. เข้า https://console.firebase.google.com → **Add project**
2. เมนู **Build → Firestore Database** → Create database → เลือก location `asia-southeast1 (Singapore)` → เริ่มในโหมด **production**
3. เมนู **Build → Authentication** → Get started → แท็บ Sign-in method → เปิด **Email/Password**
4. แท็บ **Users** → Add user → สร้างบัญชีให้เจ้าหน้าที่แต่ละคน (อีเมล + รหัสผ่าน)
5. หน้า Project Overview → ไอคอน `</>` (Web app) → ตั้งชื่อแอป → คัดลอก `firebaseConfig`

### ขั้นที่ 2 — ตั้งค่าในโปรเจกต์นี้
1. วางค่า config ลงใน `firebase/firebase-config.js`
2. Firebase Console → Firestore → แท็บ **Rules** → วางเนื้อหาจาก `firebase/firestore.rules` → **Publish**
   ⚠️ ขั้นนี้ห้ามข้าม — ไม่งั้นใครก็อ่านข้อมูลเด็กได้

### ขั้นที่ 3 — ย้ายข้อมูล (ครั้งเดียว)
1. เปิดไฟล์ `firebase/migrate.html` (ดับเบิลคลิกเปิดได้เลย ไม่ต้องรัน server)
   — ช่อง "บันทึกการทำงาน" จะตรวจความพร้อมให้อัตโนมัติ ถ้ามี ❌ ให้แก้ตามคำแนะนำก่อน
2. ล็อกอินด้วยบัญชีที่สร้างในขั้นที่ 1
3. กด **เริ่มย้ายข้อมูล** → รอจนขึ้น 🎉 ครบ 72 คน
4. ตรวจใน Firebase Console → Firestore → collection `students` ควรมี 72 documents

### ขั้นที่ 4 — สลับแอปหลักให้ใช้ Firestore ✅ ทำให้แล้ว
ไม่ต้องแก้โค้ดเอง — ระบบมีไฟล์ `js/18-firebase-bridge.js` เชื่อมให้อัตโนมัติ:

- ถ้า `firebase-config.js` **ยังเป็นค่าตัวอย่าง** → แอปทำงานแบบเดิม (localStorage) ทุกประการ
- ถ้า **ใส่ค่า config จริงแล้ว** → เปิดแอปจะเจอหน้าล็อกอินก่อน หลังล็อกอิน:
  - โหลดข้อมูลนักเรียนจาก Firestore มาแสดง
  - ถ้าคลาวด์ยังว่าง จะถามว่าอัปโหลดข้อมูลในเครื่องขึ้นไปเลยไหม
    (= ไม่ต้องใช้ migrate.html ก็ได้ ถ้าสะดวกทางนี้)
  - ทุกการแก้ไข/เพิ่ม/ลบ บันทึกขึ้นคลาวด์อัตโนมัติ
  - **realtime**: เปิดหลายเครื่องพร้อมกัน แก้ที่เครื่องหนึ่ง อีกเครื่องเห็นทันที
  - ปุ่ม "☁️ ออกจากระบบ" อยู่มุมล่างขวา
  - การดึงรายชื่อจาก Google Sheet ถูกปิด (กันข้อมูลตีกัน) แต่การส่งแบบฟอร์มไป Sheet ยังใช้ได้

### ขั้นที่ 5 — เก็บกวาดเรื่องความปลอดภัย (สำคัญที่สุด)
เมื่อยืนยันว่าแอปอ่านจาก Firestore ได้แล้ว:
1. **ลบเนื้อหาใน `js/01-data.js`** เหลือแค่ `const RAW = [];`
   → ข้อมูลเลขบัตร ปชช./บัญชีธนาคาร/ที่อยู่ของเด็กจะไม่ฝังอยู่ในโค้ดอีก
2. เพิ่มหน้าล็อกอินให้แอปหลัก (ใช้ `fb.signIn` / `fb.onAuth`)
3. ถ้า deploy ขึ้นเว็บ แนะนำ Firebase Hosting (`firebase deploy`) — ฟรีและได้ HTTPS

## 💡 หมายเหตุ
- Firestore ฟรีเทียร์ (Spark): อ่าน 50,000 ครั้ง/วัน เขียน 20,000 ครั้ง/วัน — เหลือเฟือสำหรับงานนี้
- รูปนักเรียนปัจจุบันใช้ Google Drive URL อยู่แล้ว ใช้ต่อได้เลย
  (ถ้าอยากย้ายรูปเข้า Firebase Storage ค่อยทำเป็นเฟสถัดไป)
- ระบบซิงก์ Google Sheet เดิม (`11-sheet-sync.js`) ยังทำงานคู่กันได้
  จนกว่าจะพร้อมเลิกใช้

## 🔐 ระบบบัญชีผู้ใช้และหน้า Login (ใหม่)

ไฟล์ที่เกี่ยวข้อง: `login.html`, `accounts.html`, `js/18-firebase-bridge.js` (v2), `firebase/firestore.rules` (v2)

### การไหลของระบบ
1. เปิด `index.html` โดยยังไม่ล็อกอิน → ถูกพาไป `login.html` (ดีไซน์ DLTV Fund)
2. ล็อกอินด้วย **อีเมล** หรือ **username** (เช่น dltv001) + รหัสผ่าน มีปุ่มตา 👁️ ดูรหัส,
   "จดจำฉันไว้ในระบบ", และ "ลืมรหัสผ่าน?" (ส่งลิงก์รีเซ็ตทางอีเมล)
3. ทุกการล็อกอินถูกบันทึกลง collection `loginLogs` (ใคร เข้าเมื่อไหร่)
4. ในแอปมีปุ่ม "☁️ บัญชี" มุมล่างขวา: เปลี่ยนรหัสผ่าน / จัดการบัญชี (แอดมิน) / ออกจากระบบ
5. บัญชีที่สร้างใหม่จะถูก**บังคับเปลี่ยนรหัสผ่าน**ในการเข้าครั้งแรก

### ตั้งค่าแอดมิน (ทำครั้งเดียว)
1. Firebase Console → Firestore → Start collection ชื่อ `admins`
2. Document ID = **อีเมลของคุณ** (ที่ใช้ล็อกอิน) → เพิ่มฟิลด์ `role` = `admin` → Save
3. **อัปเดต Security Rules** เป็นเวอร์ชันใหม่จากไฟล์ `firebase/firestore.rules` → Publish (สำคัญ!)

### สร้างบัญชีนักเรียนทุน 72 คน
1. ล็อกอินด้วยบัญชีแอดมิน → ปุ่ม ☁️ บัญชี → "จัดการบัญชีผู้ใช้" (หรือเปิด `accounts.html`)
2. กด "➕ สร้างบัญชีให้นักเรียนที่ยังไม่มี" — ระบบออก username `dltv001`–`dltv072`
   พร้อมรหัสเริ่มต้นสุ่ม (เช่น `Dltv4827`) ให้อัตโนมัติ
3. ตาราง "ข้อมูลการสมัคร" แสดง: ลำดับ | ชื่อ-สกุล | โรงเรียน | username | password (กดตา 👁️ ดูได้)
4. กด "⬇️ ส่งออก CSV" เพื่อพิมพ์แจกรหัสให้นักเรียน
5. หน้าเดียวกันมีตาราง "ประวัติการเข้าระบบล่าสุด" ให้ตรวจว่าใครเข้ามากรอกข้อมูลแล้วบ้าง

### สมัครสมาชิกด้วยตนเอง (register.html) — ใหม่
นักเรียนกดลิงก์ "สมัครสมาชิกนักเรียนทุน" จากหน้า login แล้วกรอก:
**ลำดับนักเรียน | ชื่อ-สกุลนักเรียนทุน | โรงเรียน | username | password** (มีตา 👁️ ดูรหัส)
ระบบจะตรวจลำดับ+ชื่อกับทะเบียนนักเรียนใน Firestore — ตรงจึงสมัครได้ และแต่ละลำดับสมัครได้ครั้งเดียว
บัญชีที่สมัครเองจะไม่ถูกบังคับเปลี่ยนรหัส (เพราะตั้งรหัสเองแล้ว) และแสดงในหน้า accounts.html ของแอดมินทันที
⚠️ ต้องใช้ Security Rules เวอร์ชันล่าสุด (ไฟล์ firestore.rules) จึงจะสมัครได้

### ปุ่มบัญชีย้ายไป sidebar (ล่างซ้าย)

- เมื่อนักเรียน**เปลี่ยนรหัสเองแล้ว** จะไม่มีใครดูรหัสได้อีก (มาตรฐานความปลอดภัย)

- ⚠️ บัญชีนักเรียนทุกคน**มองเห็นข้อมูลนักเรียนทั้งหมด**ในแอป (โครงสร้างแอปโหลดทั้งชุด)
  ถ้าต้องการให้นักเรียนเห็นเฉพาะข้อมูลตัวเอง ต้องปรับโครงสร้างเพิ่มเติม — แจ้งได้ครับ

## 🎓 อัปเดตล่าสุด: prefill แบบฟอร์ม + โหมดนักเรียน

### 1) ข้อมูลนักเรียนเติมลงแบบฟอร์ม 1/2 อัตโนมัติ
ฟิลด์ที่มีข้อมูลในระเบียนนักเรียนจะถูกเติมให้เองเมื่อเปิดฟอร์ม (แก้เพิ่มได้ตามปกติ):
ชื่อ-สกุล, ชื่อเล่น, เลขบัตร, วันเกิด, ที่อยู่, โรงเรียน+อำเภอ+จังหวัด+สังกัด, บัญชีธนาคาร,
ครูที่ปรึกษา, **ผู้ปกครอง (ชื่อ/สกุล/เบอร์)**, **ภาคเรียนล่าสุด**, **ชื่อผู้อำนวยการ**,
**ยอดเงินทุนที่ได้รับรวม** (คำนวณจากประวัติการเบิกจ่าย)
และแก้บั๊กเดิม: ฟอร์มที่เคยเปิดก่อนข้อมูลคลาวด์มาแล้วค่าว่าง "ค้างถาวร"
ตอนนี้ระบบจะเติมให้ใหม่อัตโนมัติเมื่อฟิลด์นั้นยังว่างอยู่ (ค่าที่กรอกเองไม่ถูกทับ)

### 2) โหมดนักเรียน — เห็นเฉพาะของตัวเอง
บัญชี role = student (จากการสมัครเอง หรือแอดมินสร้างให้):
- เมนูเหลือ 2 หน้า: **📝 แบบฟอร์มทุนการศึกษา** และ **📋 ติดตามการกรอกแบบฟอร์ม**
  (เข้าแอปแล้วพาไปหน้าแบบฟอร์มทันที ปุ่ม Sync/Save/Export ถูกซ่อน)
- ข้อมูลที่โหลด/เห็น/แก้ = ระเบียนของตัวเองเท่านั้น และ realtime เฉพาะเอกสารตัวเอง
- **บังคับฝั่งเซิร์ฟเวอร์ด้วย Security Rules (v4)**: ต่อให้ดัดแปลงหน้าเว็บ
  ก็อ่าน/เขียนข้อมูลนักเรียนคนอื่นไม่ได้
- ครูเห็นสิ่งที่นักเรียนกรอกแบบ realtime และแก้กลับได้ตามปกติ

### ⚠️ ต้องอัปเดต Security Rules อีกครั้ง (v4)
Firebase Console → Firestore → Rules → วางจากไฟล์ `firebase/firestore.rules` → Publish
(ถ้าไม่อัปเดต: นักเรียนจะยังเห็นข้อมูลทุกคนเหมือนเดิม)

### แก้ไข: นักเรียนเห็นรายชื่อครบ 72 คนในหน้าแบบฟอร์ม (v5)
สาเหตุ: ข้อมูลทั้งชุดที่ค้างใน cache ของเบราว์เซอร์ (จากการใช้งานครั้งก่อนในเครื่องเดียวกัน)
ถูกวาดขึ้นก่อนการกรองสิทธิ์ และหน้าแบบฟอร์ม/สถานะไม่ถูกวาดซ้ำหลังกรอง แก้แล้วดังนี้:
- ล็อกอินบัญชีนักเรียนปุ๊บ ระบบกรองข้อมูลในเครื่องเหลือของตัวเอง "ทันที" ไม่รอคลาวด์
- เขียนทับ cache ในเครื่องด้วยข้อมูลที่กรองแล้ว (ข้อมูลคนอื่นถูกล้างออกจากเครื่องนั้น)
- หน้าแบบฟอร์มทุนฯ และติดตามการกรอก ถูกวาดซ้ำอัตโนมัติทุกครั้งที่ข้อมูลเปลี่ยน
- กดปุ่ม back ของเบราว์เซอร์กลับมา → บังคับโหลดหน้าใหม่เพื่อตรวจสิทธิ์ซ้ำเสมอ (กัน bfcache)
หมายเหตุ: เครื่องที่เคยเห็น 72 คนค้างอยู่ เปิดแอปใหม่ครั้งแรกอาจเห็นแวบหนึ่งก่อนระบบล้าง cache ให้ หลังจากนั้นจะเหลือของตัวเองถาวร

### v7 — ตัด popup ตอนเข้าระบบ + เครื่องมือซ่อมการเชื่อมบัญชี
- **popup ตอนเข้าระบบถูกเอาออกทั้งหมด** (โหลดจากคลาวด์สำเร็จ ฯลฯ) เหลือเฉพาะ popup
  เกี่ยวกับ "การบันทึกข้อมูล" (บันทึกไม่สำเร็จ / อัปโหลดครั้งแรก / เปลี่ยนรหัสผ่าน)
  ข้อความสำคัญตอนเข้าระบบเปลี่ยนเป็น "แถบเหลืองด้านบน" ปิดได้ ไม่บังหน้าจอ
- **อาการนักเรียนเห็นหน้าว่าง (0 คน)**: เกิดจาก studentId ในบัญชีชี้ไปเอกสารนักเรียน
  ที่ไม่มีอยู่แล้ว (เช่น เลขบัตรถูกแก้ / ลบ-เพิ่มนักเรียนใหม่หลังสมัคร)
  → หน้า "จัดการบัญชีผู้ใช้" (accounts.html) ตรวจให้อัตโนมัติ: ขึ้นแถบแดงแจ้งจำนวน
  และมีปุ่ม **🔧 ซ่อมการเชื่อม** ท้ายแถวสีแดง กดแล้วระบบจับคู่จากลำดับ+ชื่อให้ใหม่
  เสร็จแล้วให้นักเรียนรีเฟรชแอปหรือออก-เข้าใหม่

### v8 — แก้ popup "กำลังโหลดข้อมูลจาก Google Sheet" และข้อมูลหายหลัง refresh
สาเหตุ: โค้ดเริ่มระบบ (INIT) ยังยิงโหลดข้อมูลจาก Google Sheet ทุกครั้งที่เปิดหน้า
ก่อนที่ตัวเชื่อม Firebase จะทันปิด — และเมื่อ Sheet (ซึ่งว่าง/เก่าแล้ว) ตอบกลับ
มันเขียนทับข้อมูลบนจอ ทำให้เหมือน "ข้อมูลหาย" หลัง refresh
แก้ที่ต้นทาง: เมื่อ Firebase ถูกตั้งค่า ระบบจะข้ามการโหลดจาก Google Sheet โดยสิ้นเชิง
(popup นั้นจึงไม่ขึ้นอีก และข้อมูลจากคลาวด์ไม่ถูกทับ) — การส่งออกไป Sheet ยังใช้ได้ตามเดิม

### v9 — ซ่อมการเชื่อมบัญชีนักเรียน "อัตโนมัติ" + ลดการกระพริบตอนเข้า
- นักเรียนที่บัญชีเชื่อมไม่ตรง (เห็นหน้าว่าง 0 คน): ระบบค้นหาระเบียนของตัวเอง
  จาก "ลำดับนักเรียน" แล้วผูกบัญชีใหม่ให้เองทันทีตอนเข้าระบบ — ไม่ต้องรอแอดมินกดซ่อม
  (ต้องใช้ Security Rules v5 จากไฟล์ firestore.rules — เพิ่มสิทธิ์อ่านเฉพาะเอกสาร
  ที่ "ลำดับ" ตรงกับบัญชีตัวเองเท่านั้น เพื่อการซ่อมนี้)
- ลดการกระพริบ dashboard ตอน F5 ของบัญชีนักเรียน: ระบบจำบทบาทจากครั้งก่อน
  แล้วซ่อนเมนู/สลับหน้าแบบฟอร์มทันทีตั้งแต่เปิดหน้า
- ถ้าซ่อมอัตโนมัติไม่สำเร็จ แถบเหลืองจะแสดง studentId และลำดับของบัญชี
  เพื่อให้แอดมินตรวจใน Firestore ได้ตรงจุด

### v10 — เคสเลขบัตรนักเรียนถูกแก้ไข (พบจริงจาก check.html)
อาการ: เอกสารที่บัญชีผูกไว้มีอยู่ แต่ "เลขบัตรข้างใน" เป็นคนละเลข → แอปแสดงหน้าว่าง
ระบบใหม่จัดการอัตโนมัติตอนนักเรียนเข้าแอป: อ่านเอกสารเลขใหม่ (Rules อนุญาตเพราะ
ลำดับตรงกับบัญชี) → ลบเอกสารค้างเลขเก่า → ผูกบัญชีกับเลขใหม่ → ใช้ข้อมูลจริงต่อทันที
check.html ก็วินิจฉัยเคสนี้แยกให้ชัดเจนแล้ว (กล่อง 🔁 เลขบัตรถูกแก้ไข)

### v11 — ยึด "ลำดับ+ชื่อ" เป็นตัวยืนยันตัวตน (แก้ปัญหานักเรียนเห็นผิดคน)
นโยบายใหม่ของโหมดนักเรียน:
- เอกสารที่บัญชีผูกไว้ต้อง "ชื่อตรงกับบัญชี" จึงแสดง — ถ้าไม่ตรง ระบบค้นหาตัวจริง
  จากลำดับ+ชื่อแล้วผูกใหม่ให้เอง / ถ้าหาไม่เจอ แสดงหน้าว่างพร้อมคำอธิบาย
  (ยอมว่างดีกว่าแสดงข้อมูลของคนอื่น) — ยกเว้นนามสกุลยังตรงกัน (เคสครูแก้สะกดชื่อ) จึงคงเดิม
- ระบบไม่ลบ/ย้ายเอกสารอัตโนมัติอีกต่อไป (ป้องกันข้อมูลเสียหาย)
ฝั่งแอดมิน (accounts.html):
- ปุ่มใหม่ **🩺 ตรวจ+แก้การเชื่อมทั้งระบบ** — ไล่ตรวจทุกบัญชีนักเรียน จับคู่กับทะเบียน
  ด้วยชื่อ (และลำดับ) แล้วแก้การผูกให้อัตโนมัติทั้งชุด พร้อมรายงาน:
  บัญชีที่ต้องตรวจเอง, เอกสารที่ "รหัสเอกสาร" ไม่ตรง "เลขบัตรข้างใน", และลำดับที่ซ้ำ
- การผูกบัญชีทุกจุดใช้ "รหัสเอกสารจริง" แทนฟิลด์เลขบัตร (แม่นยำแม้เลขบัตรถูกแก้)

### v12 — อุดช่องรั่วที่ทำให้ยังเห็นข้อมูลคนอื่นแม้ระบบตรวจพบว่าผูกผิด

- realtime: จะเปิดเฉพาะเมื่อการผูกผ่านการยืนยันตัวตนแล้วเท่านั้น + มียามชื่อกันอีกชั้น
- เมื่อผูกไม่สำเร็จ ระบบงดบันทึกขึ้นคลาวด์ทั้งหมด (กันเขียนทับเอกสารของคนอื่น)
- การค้นหาตัวจริงจาก "ลำดับ" ลองทั้งชนิดตัวเลขและข้อความ (ข้อมูลเก่าอาจเก็บคนละชนิด)
